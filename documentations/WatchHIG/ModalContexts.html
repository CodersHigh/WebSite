모달 시트

모달 시트는 사용자가 작업을 완료하거나, 방해받지 않고 정보를 얻거나, 메뉴 컨트롤에서의 최초 선택했던 것을 계속하는 방법을 제공한다. 모달 시트는 앱의 다른 영역과 인터렉팅 하는 것을 일시적으로 막음으로서 이런 환경을 제공한다.

앱에서 모달 경험을 최소화 하는 것이 좋다. 일반적으로 다음과 같은 경우에만 모달 컨텍스트를 고려하라:

  • 사용자의 관심을 얻는 것이 치명적으로 중요한 경우.

  • 사용자의 데이터를 애매한 상태로 남겨두지 않기 위해 내장된 작업이 완료되거나-명백히 버려져야 하는 경우.

모달 인터페이스는 하나의 화면을 사용하거나 페이지-기반 레이아웃을 이용해 여러개의 화면을 표시한다. 이 두가지 인터페이스의 유일한 차이점은 페이지-기반 인터페이스에만 바닥영역에 추가되는 점 인디케이터이다.

모달 인터페이스의 좌상단은 닫기 버튼으로 예약되어 있다. 사용자가 이 버튼을 탭하면(또는 왼쪽 모서리 스와이프를 하면), 시스템은 더 이상의 액션 없이 모달 인터페이스를 제거한다. 이 버튼을 두는 것은 의무사항이지만 의미를 전달하기 위해 텍스트를 변경하는 것은 가능하다. 콘텐트의 본문안에 추가적인 닫기 버튼을 더하지 마라. 닫기 버튼의 전형적인 타이틀은 "닫기" 또는 "취소"이다. 닫기 버튼의 색상은 항상 흰색이다.

작업이 승인을 필요로 한다면, 모달 인터페이스의 본문안에 승인 버튼을 제공하라. 승인 버튼에는 표준 버튼을 사용하라. 이 버튼을 탭하면 적당한 액션을 수행한 뒤 모달 인터페이스를 제거해야 한다.

모달 작업을 단순하게 유지하라. 하나의 모달 인터페이스 위에 두번 째 모달 인터페이스를 표시하는 것을 삼가라.
