레이아웃

레이아웃 가이드라인

인터페이스에서 연속된 컨트롤의 개수를 제한하라. 버튼을 연속적으로 배치할 때, 버튼들에는 텍스트 대신 아이콘을 사용하라. 세 개 이상의 아이템들을 연속적으로 배치하지 마라. 아이템들이 많을수록 사용자가 타겟을 탭하기가 더 어려워진다.

화면 너비 전체를 사용하라. 애플 와치의 베젤이 콘텐트에 대한 시각적 패딩을 제공하기 때문에, 화면의 모서리와 당신의 콘텐트 사이에 마진을 포함시킬 필요는 없다. (iOS 시뮬레이터에서는 이 패딩이 나타나지 않는다.)

앱은 아이템들에 대해 연관성있는 위치를 사용한다. 앱이 애플 와치의 서로 다른 사이즈에서 동일한 인터페이스를 표시하기 때문에, 연관성 있게 위치를 잡으면, 필요에 따라 사용가능한 공간을 채우면서 펼쳐질 수 있다.

레이아웃시에 왼쪽 정렬을 선호하라. 인터페이스의 요소들은 위에서 아래로, 오른쪽에서 왼쪽으로 배치된다. 왼쪽 모서리부터 요소들을 정렬하는 것은 넓혀질 공간을 확보하고 콘텐트를 전체적으로 표시하게 한다.

문자 버튼들을 전체 너비로 하라. 텍스트 레이블을 표시하는 버튼들은 전체 너비로 만들어서 전체 버튼 레이블이 항상 보이도록 해야 한다.

컨텍스트 메뉴를 이용해서 부가적인 액션을 제공하라. 인터페이스에 버튼을 더하는 것 보다, 덜 자주 사용하는 액션에 대해서는 컨텍스트 메뉴를 이용하라. 보다 자세한 정보는 메뉴를 보라.

화면 크기

애플 와치에 표시하는 콘텐트는 장치의 화면 크기에 상관없이 동일해야 한다. 레이아웃을 디자인 할 때, 가능한 공간 안에서 자연스럽게 아이템들이 펼쳐지거나 좁혀지도록 한다.

필요하다면 다른 화면 크기를 위해 이미지 에셋들을 제공하라. 양쪽 화면 크기에서 모두 보기 좋다면 동일한 이미지 리소스를 사용하라; 아니면, 각 화면 크기를 위해 각기 다른 이미지 리소스를 제공하라.
