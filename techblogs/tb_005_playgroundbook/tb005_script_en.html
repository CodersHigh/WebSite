Create a “Playground Book”

Posted on March 18, 2018. | Author: SeongkwanYoon

Apple has announced the WWDC 18 Scholarship Application Guide.
Continued from 2017, this year's Scholarship will also be chosen rated based on Playground Book.
This tutorial is provided based on the experience of creating a playground book at Coders High.
I hope that it will help many students get the WWDC Scholarship.
The 2018 WWDC Scholarship application is available from March 26 through April 1.




1. Getting Started

Playground Books is a book content used in the iPad's Swift Playground app. In 2016, Apple created the Swift Playground app to use the Swift language for coding training and distributed LearnToCode 1, 2, 3




a. Materials

For now, you can only write a playground book on your Mac's Xcode, and you'll need to put it in the Swift playground on your iPad device to see the output. 
Because there is no Swift Playground app on iPhone, there is no way to check the content, and also the iPad provided by the simulator can not be used to check the results since there is no Swift Playground app.
In other words, you first need to prepare this:

  • Mac with Xcode 9 or later installed
  • IPad for viewing results

And it's convenient to have an iCloud account ready to automatically link the results




b. Preparation of mind

As mentioned above, making playground books is difficult. 
You should have a good understanding of the Swift language and be able to handle Xcode at this level by completing your app.
While Google can search for a few pages of code that you can paste when you type in keywords in an app development, Playground Books does not have a guide document or tutorial, except for a reference document and a few pieces of video. Even if you do a Google search, it hardly comes out. (This is why I wrote this article) 
Fortunately, there are a variety of contents Apple has created over the past two years, so if you hack into them, you'll be able to create the Playstation book I would like to express.

d. Official Starter Kit

You can download the Playground Book Starter Kit from Apple.

You can check the inside of the bundle by clicking the Show Package Contents button in the Context menu (right click) of the Starter.playgroundbook file.

f you look at the stack of starter kits, you will find a .playgroundchapher folder in the Chapter folder and two Pages folders inside it. A structure with two pages in one chapter.


Double-click the Content.swift file in the Page1.playgroundpage file to open Xcode's editor and modify its contents.

Folder configuration
  • .playgroundbook : The top-level folder. Book bundle
  • .playgroundchapter : chapter folder
  • .playgroundpage : individual page folder
  • .cutscenepage : Cut scene (chapter start animation) Page folder

Manifest file
At each stage, there is a plist (link) format manifest file that defines the properties of the content. You must modify the manifest file whenever you modify content, such as adding a page or a chapter. 
The page's manifest has a hint entry, which also provides a hint to the user. 
Our playground book is an app with one chapter and one page. The .playgroundbook manifest must contain exactly one chapter name, and the .playgroundchapter manifest must contain exactly one page name.




e. Screen Layout

The playground book is divided into three areas as shown below.

LiveView : This is where you can see the results of the code execution. The contents of the LiveView.swift file appears here.



Prose : This is where a description content of the page enters. The content of the Content.swift file appears here.



Code : This is where you can write your code. The content of the Content.swift file appears here.

Now let's create a working environment before creating playground book content. To run your code, you need to put the playground book file into the Swift Playground app on your iPad. 
There are two ways to copy files from your Mac to your iPad.

  • After you set up iPad to share SwiftPlayground with iCloud, modify the files in that folder in iCloud on your Mac and sync them.
  • You can open it in the SwiftPlayground app by selecting the playground file on your Mac and copying it to your iPad with AirDrop.

While the Swift files used in the Playground book import UIKit, since the method's auto-completion proposal is not working properly, it is recommended to create an iOS App after creating a complicated content.

c. Creating Prose

Prose is a place to describe the quizzes presented on the page. 
You can put text that is shaped in a markup-like syntax into the playground book.

This is described in detail in the Page Annotation Overview document and Markup Formatting Reference in the previous Playground Book Format Reference document.
Here's how to create a minimal Prose.

Starting from /*: to */ is the area of playground markup. 
If you use #, it’s Heading 1, and ## will make Heading 2. As the number of # increases, it becomes smaller and smaller. 
After leaving out a line, write down more than 4 spaces to will make it recognized as a code.
I did not deal with it now, but if you put a number in front, a numbered list is created, and if you put a * before, a bullet list will be created.

Since Playground Books is content for Swift language learning, consumers of this content may not yet be familiar with the Swift language. 
Therefore, instead of showing all the code, you need to show only what the learner is going to focus on and cover the rest.

PlaygroundPage class
This class represents the individual pages of the playground and manages basic operations. 
You can access the current page in a singleton format like PlaygroundPage.current. For example, if you want to check whether the user entered a particular string in the code area of the current page, check whether PlaygroundPage.current.text has a particular string. 
PlaygroundPage.current.liveview provides access to live view proxy. 
This is a somewhat difficult part, and you cannot directly call the live view method because it is accessed through the proxy, rather than directly accessing the live view. 
You must send the PlaygroundRemoteLiveViewProxy object in the middle to send and receive messages. Details are provided below.

So the code above assigns the current playground page to a constant called “page”, and assigns a proxy for the live view to a constant called “proxy”.  
Then, we will use the page constant to determine whether to proceed with the playground according to the user input, that is, to check the input value and pass it to the next page or provide a hint about the wrong input value.
Live view update according to user input will pass through the proxy.

PlaygroundRemoteLiveViewProxy Class
An object that enables the transmission of messages between live views and playground pages. The live view object currently active on the playground page must be downcasted to PlaygroundRemoteLiveViewProxy to exchange messages. 
In other words, for the page you are creating, you can not cast the HelloViewController directly to live view in Content.swift and call the method. So downcast it to PlaygroundRemoteLiveViewProxy, access it, assign an object to PlaygroundRemoteLiveViewProxyDelegate, and call remoteLiveViewProxy (_ :, received :) Function to receive messages from live view and send messages to the page using the receive (_ :) and send (_ :) functions on objects that conform to the PlaygroundLiveViewMessageHandler protocol.


The setName (: _) function passes the username to PassPort.shared.name. 
PassPort classcreates a typical Singleton accessible object. This is because passports required for boarding should be accessible from all pages as one object of the same.

f. Create a user input area

Users should write a code to solve the problems on each page in the coding area created by the author in the Playstation book content.
Authors can adjust the solvency of the solution by allowing the user to fill in some of the code they have exposed, in addition to the code they have hidden.


There is a space between the // #-editable-code markup and // # -end-editable-code markup where the user can enter the code.

You can give a suggestion for code completion through // #-code-completion markup. The iPad is inconvenient for keyboard input, so it will be nice to provide some pre-input code for the learner's efficiency. 
This page hides all of the basic code suggestions through // #-code-completion (everything, hide) markup and then uses //#-code-completion(identifier, show, setName(_:)) markup to provide the setName () function to the user.




g. Configuring Live View

Previously, we connected HelloViewController to live view of PlaygroundPage.current which we are working on now. 
Therefore, the content to be displayed in Live View can be configured in the same way as configuring UI of iOS app in HelloViewController.

iIt's a little different than creating an iOS app's UI, but if you have experience creating an app, you'll be able to organize your view without much difficulty.
Create a mainView with an object of UIVIew, and create a Label and an ImageView to construct the screen. 
After you set the font size, font, color, text, etc. of the label, you can display the label on the screen by putting it into the subview of mainView and applying auto-layout. 
The same is true for ImageView, where you can create a Resources folder in the .playgroundpage folder or a global Resources folder in the .playgroundbook folder.





h. Checking user input values

The playground page can change the LiveView according to the user's input value or proceed to the next page. 
Of course, if the input is wrong, you can give the correct answer by giving the proper hint to the user.


The playground page uses the enum property called AssessmentStatus to display the task success text of the page or provide additional hints. 
AssessmentStatus consists of

  • case fail(hints: [String], solution: String?)
  • case pass(message: String?)
And the message at the time of .pass is displayed in a dialogue box after executing the code. 
Here we analyze the text of the playground page, make sure that the function name setName is used, and then 
jump to the next page using the [Orientation](@ next) markup in a message to celebrate success.
